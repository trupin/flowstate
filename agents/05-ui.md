# Agent 5: Web UI — React Frontend

## Role

You are implementing the Flowstate web frontend: a React + TypeScript application with graph visualization, real-time log streaming, and flow control capabilities.

Read `specs.md` section **10 (Web Interface)** for UI requirements, **10.3 (WebSocket Protocol)** for the real-time event protocol, **10.7 (Sidebar)** for navigation, **10.8 (File Watcher)** for live reload behavior, and **10.9 (Start Run Modal)** for the run launch UX.

## Phase

**Phase 1 (scaffold + real backend)** — no mock data. Build the UI components against the real FastAPI backend. The Vite dev server proxies `/api` and `/ws` routes to the backend.

**Phase 3 (integration polish)** — end-to-end testing with the full system.

## Sidebar (`Sidebar.tsx`)

Three collapsible sections. See specs.md Section 10.7.

- **FLOWS**: Lists all discovered `.flow` files from the watched directory. Shows validity status (green dot = valid, hollow dot = errors). Click navigates to Flow Library with that flow selected.
- **ACTIVE RUNS**: Lists currently running/paused flows with status color indicator. Click navigates directly to Run Detail.
- **SCHEDULES**: Lists configured recurring flows with next trigger time.

The sidebar subscribes to WebSocket events for real-time updates:
- `flow.file_changed` / `flow.file_error` / `flow.file_valid` → update flow list validity indicators
- `flow.started` / `flow.status_changed` / `flow.completed` → update active runs list

## Page Specifications

### Flow Library (`FlowLibrary.tsx`)

- Lists all discovered `.flow` files from the watched directory (via `GET /api/flows`)
- Each flow shows: name, validity status, last modified time
- Click a flow → right panel shows graph preview (read-only React Flow visualization of the parsed AST)
- If the selected flow has parse/type-check errors: show a persistent **error banner** (not a toast) with the error details, and preserve the last valid graph
- **No DSL editor, no upload, no delete** — flows are managed as files on disk
- "Start Run" button on valid flows → opens Start Run Modal

### Run Detail (`RunDetail.tsx`)

The main page. Three panels (specs.md Section 10.1):

- **Graph View** (~60% width): React Flow graph with live status updates
- **Log Viewer** (~40% width): Raw streaming logs for selected node
- **Control Panel** (bottom bar): Pause/resume/cancel/retry + budget bar

- Subscribe to WebSocket on mount, unsubscribe on unmount
- Graph updates in real-time as events arrive
- Click a node → log viewer shows that task's output
- Control buttons send actions via WebSocket or REST

## Graph Visualization (`GraphView.tsx`)

Use React Flow (`@xyflow/react`) with dagre for automatic layout.

### Node rendering — hybrid compact/expanded (specs.md Section 10.4)

**Default (compact pill):**
- Pill shape with node name + status color fill
- Small enough that large graphs remain readable

**Click to expand:**
- Shows: type badge (entry/task/exit), generation count, elapsed time, cwd
- Expands in-place without disrupting graph layout

**Hover tooltip:**
- Quick info (status, type, generation) without expanding

### Node colors

| Status | Color | Effect |
|--------|-------|--------|
| Pending | `#9CA3AF` (gray) | Static |
| Waiting | `#A855F7` (purple) | Countdown timer |
| Running | `#3B82F6` (blue) | Animated pulse |
| Completed | `#22C55E` (green) | Static |
| Failed | `#EF4444` (red) | Static |
| Skipped | `#F97316` (orange) | Static |
| Paused | `#EAB308` (yellow) | Static |

### Edge rendering
- Unconditional: solid arrow
- Conditional: dashed arrow with truncated `when` text as label
- Fork: solid arrows diverging from one node
- Join: solid arrows converging to one node
- Active edge (currently being traversed): highlighted/animated

### Layout
- Use dagre with `rankdir: 'TB'` (top-to-bottom)
- Recalculate layout when graph structure changes

## Log Viewer (`LogViewer.tsx`)

Raw streaming output — specs.md Section 10.5.

- Displays raw output for the currently selected task (clicked node in graph)
- Receives `task.log` events via WebSocket
- **No structured parsing** of tool_use/tool_result — show everything as raw text
- Monospace font, dark background
- Auto-scroll to bottom with pin/unpin toggle
- "Clear" button (client-side only)

## Control Panel (`ControlPanel.tsx`)

- **Pause** button: visible when status is `running`. Sends `pause` action via WS.
- **Resume** button: visible when status is `paused` or `budget_exceeded`. Sends `resume` via REST (POST /api/runs/:id/resume).
- **Cancel** button: visible when status is `running` or `paused`. Confirms first.
- **Retry** button: visible when a failed task is selected. Sends `retry_task` via WS.
- **Skip** button: visible when a failed task is selected. Sends `skip_task` via WS.
- **Budget bar**: progress bar showing elapsed_seconds / budget_seconds. Changes color at 75% (yellow), 90% (orange), 95%+ (red).

## Start Run Modal (`StartRunModal.tsx`)

See specs.md Section 10.9.

- Triggered by "Start Run" button on Flow Library
- Shows: flow name, parameter form (auto-generated from `param` declarations in the DSL)
- Type-appropriate controls:
  - `string` → text input
  - `number` → number input
  - `bool` → checkbox
- Default values pre-filled from DSL
- "Start" button calls `POST /api/flows/:id/runs` and navigates to Run Detail

## Error Banner (`ErrorBanner.tsx`)

- Persistent banner (not a toast) shown when a `.flow` file has parse/type-check errors
- Displays error details (line numbers, messages)
- Dismissable, but reappears on next error
- Sits above the graph preview area

## API Client (`client.ts`)

Simple fetch wrapper:

```typescript
const api = {
    flows: {
        list: () => get<DiscoveredFlow[]>('/api/flows'),
        get: (id: string) => get<DiscoveredFlow>(`/api/flows/${id}`),
    },
    runs: {
        list: (status?: string) => get<FlowRun[]>(`/api/runs${status ? `?status=${status}` : ''}`),
        get: (id: string) => get<FlowRunDetail>(`/api/runs/${id}`),
        start: (flowId: string, req: StartRunRequest) => post<{ id: string }>(`/api/flows/${flowId}/runs`, req),
        pause: (id: string) => post(`/api/runs/${id}/pause`),
        resume: (id: string) => post(`/api/runs/${id}/resume`),
        cancel: (id: string) => post(`/api/runs/${id}/cancel`),
        retryTask: (runId: string, taskId: string) => post(`/api/runs/${runId}/tasks/${taskId}/retry`),
        skipTask: (runId: string, taskId: string) => post(`/api/runs/${runId}/tasks/${taskId}/skip`),
        taskLogs: (runId: string, taskId: string, after?: string) =>
            get<LogEntry[]>(`/api/runs/${runId}/tasks/${taskId}/logs${after ? `?after=${after}` : ''}`),
    },
    schedules: {
        list: () => get<FlowSchedule[]>('/api/schedules'),
        pause: (id: string) => post(`/api/schedules/${id}/pause`),
        resume: (id: string) => post(`/api/schedules/${id}/resume`),
        trigger: (id: string) => post(`/api/schedules/${id}/trigger`),
    },
};
```

## TypeScript Types (`types.ts`)

Define types matching the REST API responses and WebSocket events:

```typescript
interface DiscoveredFlow {
    id: string;
    name: string;
    file_path: string;
    source_dsl: string;
    is_valid: boolean;
    errors: FlowError[];
    params: FlowParam[];
    last_modified: string;
}
interface FlowError { line: number; column: number; message: string; rule?: string; }
interface FlowParam { name: string; type: 'string' | 'number' | 'bool'; default_value?: string | number | boolean; }
interface FlowRun { id: string; status: FlowRunStatus; elapsed_seconds: number; budget_seconds: number; ... }
interface TaskExecution { id: string; node_name: string; status: TaskStatus; generation: number; ... }
interface EdgeTransition { id: string; from_node: string; to_node: string; edge_type: string; ... }
interface LogEntry { id: number; task_execution_id: string; log_type: string; content: string; timestamp: string; }
interface FlowEvent { type: string; flow_run_id: string; timestamp: string; payload: Record<string, unknown>; }

type FlowRunStatus = 'created' | 'running' | 'paused' | 'completed' | 'failed' | 'cancelled' | 'budget_exceeded';
type TaskStatus = 'pending' | 'running' | 'completed' | 'failed' | 'skipped';
```
