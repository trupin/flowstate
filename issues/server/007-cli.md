## Summary
Implement the full Flowstate CLI using typer, replacing the placeholder from SHARED-002. The CLI provides seven commands: `check` (parse + type-check a `.flow` file), `server` (start the web server), `run` (execute a flow from a file), `runs` (list all runs), `status` (show run details), `schedules` (list schedules), and `trigger` (manually trigger a scheduled flow). The `check` and `server` commands can work without the engine; the `run`, `runs`, `status`, `schedules`, and `trigger` commands depend on the server being available or on direct DB/engine access.

## Acceptance Criteria
- [ ] `src/flowstate/cli.py` is fully implemented with all 7 commands
- [ ] Entry point `flowstate` is registered in `pyproject.toml` as `flowstate.cli:app`
- [ ] `flowstate --help` prints help with all commands listed
- [ ] `flowstate check <file.flow>`:

  - On success: prints "OK" and exits 0
  - On parse error: prints the error(s) and exits 1
  - On type-check errors: prints each error and exits 1
  - Non-existent file: prints error and exits 1
- [ ] `flowstate server`:
  - Starts the FastAPI server via uvicorn
  - Accepts `--host` (default from config), `--port` (default from config), `--config` (path to TOML)
  - Prints "Starting Flowstate server on <host>:<port>"
- [ ] `flowstate run <file.flow>`:
  - Accepts `--param key=value` (repeatable)
  - Parses and validates the flow file
  - Starts a run (either directly via engine or via HTTP to a running server)
  - Prints the flow_run_id and exits

  - Lists all runs with: ID (truncated), flow name, status, started time

- [ ] `flowstate status <run-id>`:
  - Shows run details: status, flow name, elapsed time, budget
  - Lists task executions with status
  - Accepts partial run-id prefix (matches uniquely)
- [ ] `flowstate schedules`:
  - Lists all schedules with: flow name, cron expression, status, next run time
- [ ] `flowstate trigger <flow-name>`:
  - Triggers a manual run of a scheduled flow
  - Prints the new flow_run_id
- [ ] All tests pass: `uv run pytest tests/server/test_cli.py`

#### CLI Structure

```python
import sys
from pathlib import Path
from typing import Annotated, Optional

import typer

app = typer.Typer(
    name="flowstate",
    help="State-machine orchestration system for AI agents.",
    no_args_is_help=True,
)


@app.command()
def check(
    path: Annotated[str, typer.Argument(help="Path to a .flow file")],
) -> None:
    """Parse and type-check a .flow file."""
    file_path = Path(path)
    if not file_path.exists():
        typer.echo(f"Error: File not found: {path}", err=True)
        raise typer.Exit(code=1)

    source = file_path.read_text()

    from flowstate.dsl.parser import parse_flow
    from flowstate.dsl.type_checker import check_flow

    try:
        flow_ast = parse_flow(source)
    except Exception as e:
        typer.echo(f"Parse error: {e}", err=True)
        raise typer.Exit(code=1)

    errors = check_flow(flow_ast)
    if errors:
        for error in errors:
            typer.echo(f"Type error: {error}", err=True)
        raise typer.Exit(code=1)

    typer.echo("OK")


@app.command()
def server(
    host: Annotated[Optional[str], typer.Option(help="Server host")] = None,
    port: Annotated[Optional[int], typer.Option(help="Server port")] = None,
    config: Annotated[Optional[str], typer.Option("--config", help="Path to config file")] = None,
) -> None:
    """Start the Flowstate web server."""
    import uvicorn

    from flowstate.config import load_config
    from flowstate.server.app import create_app

    cfg = load_config(path=config)

    # CLI flags override config file
    final_host = host or cfg.server_host
    final_port = port or cfg.server_port

    app = create_app(config=cfg)

    typer.echo(f"Starting Flowstate server on {final_host}:{final_port}")
    uvicorn.run(app, host=final_host, port=final_port)


@app.command()
def run(
    path: Annotated[str, typer.Argument(help="Path to a .flow file")],
    param: Annotated[Optional[list[str]], typer.Option("--param", help="Parameter as key=value")] = None,
    config: Annotated[Optional[str], typer.Option("--config", help="Path to config file")] = None,
) -> None:
    """Start a flow run from a .flow file."""
    file_path = Path(path)
    if not file_path.exists():
        typer.echo(f"Error: File not found: {path}", err=True)
        raise typer.Exit(code=1)

    # Parse params from key=value pairs
    params: dict[str, str] = {}
    for p in (param or []):
        if "=" not in p:
            typer.echo(f"Error: Invalid param format '{p}'. Use key=value.", err=True)
            raise typer.Exit(code=1)
        key, _, value = p.partition("=")
        params[key] = value

    source = file_path.read_text()

    from flowstate.dsl.parser import parse_flow
    from flowstate.dsl.type_checker import check_flow

    try:
        flow_ast = parse_flow(source)
    except Exception as e:
        typer.echo(f"Parse error: {e}", err=True)
        raise typer.Exit(code=1)

    errors = check_flow(flow_ast)
    if errors:
        for error in errors:
            typer.echo(f"Type error: {error}", err=True)
        raise typer.Exit(code=1)

    # Start the run
    import asyncio

    from flowstate.config import load_config

    cfg = load_config(path=config)

    async def _run() -> str:
        from flowstate.engine.executor import FlowExecutor
        from flowstate.state.repository import FlowstateDB

        db = FlowstateDB(cfg.database_path)
        import uuid
        run_id = str(uuid.uuid4())
        executor = FlowExecutor(
            flow_run_id=run_id,
            flow_ast=flow_ast,
            params=params,
            db=db,
            config=cfg,
        )
        await executor.execute()
        return run_id

    run_id = asyncio.run(_run())
    typer.echo(f"Run completed: {run_id}")


@app.command()
def runs(
    status: Annotated[Optional[str], typer.Option(help="Filter by status")] = None,
    config: Annotated[Optional[str], typer.Option("--config", help="Path to config file")] = None,
) -> None:
    """List all flow runs."""
    from flowstate.config import load_config
    from flowstate.state.repository import FlowstateDB

    cfg = load_config(path=config)
    db = FlowstateDB(cfg.database_path)
    all_runs = db.list_runs(status=status)

    if not all_runs:
        typer.echo("No runs found.")
        return

    # Table header
    typer.echo(f"{'ID':<12} {'Flow':<20} {'Status':<12} {'Started':<20}")
    typer.echo("-" * 64)
    for r in all_runs:
        short_id = r.id[:8] + "..."
        typer.echo(f"{short_id:<12} {r.flow_name:<20} {r.status:<12} {r.started_at}")


@app.command()
def status(
    run_id: Annotated[str, typer.Argument(help="Run ID (or prefix)")],
    config: Annotated[Optional[str], typer.Option("--config", help="Path to config file")] = None,
) -> None:
    """Show detailed status of a flow run."""
    from flowstate.config import load_config
    from flowstate.state.repository import FlowstateDB

    cfg = load_config(path=config)
    db = FlowstateDB(cfg.database_path)

    # Support prefix matching
    run = db.get_run(run_id)
    if not run:
        # Try prefix match
        all_runs = db.list_runs()
        matches = [r for r in all_runs if r.id.startswith(run_id)]
        if len(matches) == 0:
            typer.echo(f"Error: Run '{run_id}' not found.", err=True)
            raise typer.Exit(code=1)
        elif len(matches) > 1:
            typer.echo(f"Error: Ambiguous run ID prefix '{run_id}'. Matches:", err=True)
            for m in matches:
                typer.echo(f"  {m.id}", err=True)
            raise typer.Exit(code=1)
        run = matches[0]

    typer.echo(f"Run: {run.id}")
    typer.echo(f"Flow: {run.flow_name}")
    typer.echo(f"Status: {run.status}")
    typer.echo(f"Elapsed: {run.elapsed_seconds:.1f}s / {run.budget_seconds}s")
    typer.echo()

    tasks = db.get_task_executions(run.id)
    if tasks:
        typer.echo("Tasks:")
        for t in tasks:
            typer.echo(f"  {t.node_name} (gen {t.generation}): {t.status}")


@app.command()
def schedules(
    config: Annotated[Optional[str], typer.Option("--config", help="Path to config file")] = None,
) -> None:
    """List all flow schedules."""
    from flowstate.config import load_config
    from flowstate.state.repository import FlowstateDB

    cfg = load_config(path=config)
    db = FlowstateDB(cfg.database_path)
    all_schedules = db.list_schedules()

    if not all_schedules:
        typer.echo("No schedules found.")
        return

    typer.echo(f"{'Flow':<20} {'Cron':<20} {'Status':<10} {'Next Run':<20}")
    typer.echo("-" * 70)
    for s in all_schedules:
        next_run = s.next_run_at or "—"
        typer.echo(f"{s.flow_name:<20} {s.cron_expression:<20} {s.status:<10} {next_run}")


@app.command()
def trigger(
    flow_name: Annotated[str, typer.Argument(help="Flow name to trigger")],
    config: Annotated[Optional[str], typer.Option("--config", help="Path to config file")] = None,
) -> None:
    """Manually trigger a scheduled flow."""
    import asyncio

    from flowstate.config import load_config
    from flowstate.state.repository import FlowstateDB

    cfg = load_config(path=config)
    db = FlowstateDB(cfg.database_path)
    all_schedules = db.list_schedules()
    schedule = next((s for s in all_schedules if s.flow_name == flow_name), None)

    if not schedule:
        typer.echo(f"Error: No schedule found for flow '{flow_name}'.", err=True)
        raise typer.Exit(code=1)

    # Load and execute the flow
    async def _trigger() -> str:
        from flowstate.engine.executor import FlowExecutor
        import uuid
        run_id = str(uuid.uuid4())
        # ... load flow AST from registry or file ...
        # ... create executor and execute ...
        return run_id

    run_id = asyncio.run(_trigger())
    typer.echo(f"Triggered: {run_id}")


if __name__ == "__main__":
    app()
```

#### Import Strategy

Use lazy imports (inside functions) for heavy dependencies like `flowstate.dsl.parser`, `flowstate.engine.executor`, `flowstate.state.repository`, and `uvicorn`. This keeps `flowstate --help` fast — typer only needs to know the command signatures, not load the entire dependency tree.

#### Parameter Parsing for `run` Command

The `--param` flag is repeatable: `flowstate run my.flow --param focus=auth --param depth=3`. Each value is a string; the engine is responsible for type coercion based on the flow's declared param types.

#### Run ID Prefix Matching for `status` Command

The `status` command accepts a full UUID or a unique prefix. It first tries exact match via `db.get_run(run_id)`, then falls back to scanning all runs for prefix matches. If the prefix is ambiguous (matches multiple runs), it lists the matches and exits with code 1.

### Edge Cases
- `flowstate check` on a file with multiple errors: print all errors, not just the first one.
- `flowstate run` with no `--param` flags: works if the flow has no required parameters.
- `flowstate run` with params that contain `=` in the value: use `partition("=")` not `split("=")` to correctly handle `--param query="a=b"`.
- `flowstate server` with a port already in use: uvicorn will print its own error message and exit with non-zero status.
- `flowstate status` with a UUID prefix that matches nothing: print error, exit 1.
- `flowstate runs` with no DB yet (`~/.flowstate/flowstate.db` doesn't exist): the DB should be created automatically, and the empty table returns no runs. If the DB module doesn't auto-create, catch the error and print "No runs found."
- `flowstate trigger` for a flow that has no schedule: print error, exit 1.

## Testing Strategy

Create `tests/server/test_cli.py`. Use typer's `CliRunner` for testing CLI commands.

1. **test_help** — Invoke `flowstate --help`. Verify exit code 0 and that all 7 commands are listed.

2. **test_check_valid_file** — Write a valid `.flow` file to a tmp path. Run `flowstate check <path>`. Verify output is "OK" and exit code 0.

3. **test_check_parse_error** — Write an invalid `.flow` file. Run `flowstate check <path>`. Verify exit code 1 and error message in output.

4. **test_check_type_error** — Write a `.flow` file that parses but has type errors (e.g., no entry node). Verify exit code 1 and type error message(s).

5. **test_check_file_not_found** — Run `flowstate check nonexistent.flow`. Verify exit code 1 and "File not found" in output.

6. **test_server_command_starts** — Mock `uvicorn.run`. Run `flowstate server`. Verify `uvicorn.run` was called with the correct host and port from default config.

7. **test_server_custom_port** — Mock `uvicorn.run`. Run `flowstate server --port 9090`. Verify the port 9090 is used.

8. **test_runs_empty** — Mock the DB to return no runs. Run `flowstate runs`. Verify "No runs found." output.

9. **test_runs_with_results** — Mock the DB to return 2 runs. Verify table output.

10. **test_runs_filter_by_status** — Mock the DB. Run `flowstate runs --status running`. Verify the DB was queried with the status filter.

11. **test_status_by_full_id** — Mock DB with a run and tasks. Run `flowstate status <full-uuid>`. Verify detailed output.

12. **test_status_by_prefix** — Mock DB with runs. Run `flowstate status <first-8-chars>`. Verify the correct run is shown.

13. **test_status_ambiguous_prefix** — Mock DB with two runs sharing a prefix. Verify exit code 1 and "Ambiguous" message.

14. **test_run_with_params** — Mock the parser and executor. Run `flowstate run test.flow --param focus=auth --param depth=3`. Verify params are correctly parsed.

Use `typer.testing.CliRunner` and mock the DSL parser, DB, and engine as needed. The CLI tests should be fast (no actual server startup, no real DB access).
